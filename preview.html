10/ Do & Don't Enforced by the Quality Evaluator

DO

  • Use weight and luminance for hierarchy, not color
  • Set data in mono with tabular-nums
  • Reach for 1px border before shadow
  • Keep motion under 300ms
  • Start entry from scale(0.95), never scale(0)
  • Make exits ~20% faster than entries
  • Set transform-origin explicitly on popovers
  • Use … for truncation, not ...
  • Cap body at 65ch

DON'T

  • Write transition: all
  • Use a shadow with blur ≥ 25px (the AI-slop shadow)
  • Transition color on hover (opacity instead)
  • Use Inter, Roboto, system-ui as the primary face
  • Color a CTA button with accent-live
  • Put two live dots with different meanings in one viewport
  • Apply gradient fills to buttons, cards, or backgrounds
  • Use radius above lg on a dark card
  • Stretch body text full viewport width
