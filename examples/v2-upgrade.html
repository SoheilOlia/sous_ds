CONTENT DESIGN

How the system writes.

The voice is terse instrument-readout: present tense, active subject, numerals in mono, no narration of the work itself. Below — the voice in context, not as a corrections list.

HEADLINE

Ten stages, four wired.

Verb-led. Present tense. ≤7 words. Numerals run in mono inside the headline (the alpha/numeric mix).

STATE CALLOUT

Amber: diagnostics tightened the bar.

State word, colon, verb-led clause. The state word names what's true; the clause names why.

BODY

Bar confidence climbs to 78–80% after #16 and #17 land. Green requires PR #20 (CI gate).

First sentence ≤12 words names the state. Second ≤24 words names cause or next gate. Numerals in mono. No third sentence without structural reason.

RECEIPT ROW

[#17] PR retired stale handoffs. [MERGED] 2026-04-29

[ID] · subject-verb-object · [STATE] · timestamp. Mono throughout. Body never carries file paths — those live in <Citation>.

SEVEN RULES
  1. Terse-first sentences. ≤12 words to open.
  2. Present tense, active voice. Subject acts.
  3. Numerals, IDs, timestamps in mono.
  4. File paths never in body prose.
  5. Project jargon gets a one-clause unpack on first appearance.
  6. Rhythm variance. No two adjacent sections share micro-template.
  7. No status-meeting voice. The system narrates state, not its own existence.
